# Architecture Decision Records

This directory contains Architecture Decision Records (ADRs) for the MuMain project.

## What is an ADR?

An ADR captures an architecturally significant decision along with its context and consequences. ADRs are numbered sequentially and are immutable once accepted — superseded decisions are marked as such with a link to the replacement.

## Creating a New ADR

1. Copy `template.md` to `NNNN-short-title.md` (next sequential number)
2. Fill in all sections
3. Submit as part of the PR that implements the decision
4. Update this index
